# EI Fragment Calculator v1.9.2 - Release Notes

**Release Date:** April 17, 2026  
**Status:** ✅ Production Ready

---

## Overview

Version 1.9.2 is a critical bug fix release that resolves three essential GUI visualization issues introduced in v1.9.1. The Fragment Calculator tab's multi-view layout (compound list, spectrum canvas, peak results table) now functions correctly with proper spectrum rendering and peak table population.

---

## Bug Fixes

### 1. ✅ Spectrum Visualization Not Displaying

**Issue:** Spectrum chart was not rendering in the Fragment Calculator tab when users selected compounds from the compound list.

**Root Cause:** The `_render_spectrum()` method was preceded by a premature `_cleanup_spectrum()` call that closed the matplotlib figure before rendering could occur. This caused a silent failure—no error was logged, but the spectrum remained blank.

**Fix:** Removed the premature cleanup call from the compound selection event handler. Spectrum now renders correctly on compound selection.

**Impact:** Users can now see visual representation of the input mass spectrum when selecting compounds.

---

### 2. ✅ Peak Table Not Clearing at Run Start

**Issue:** When users clicked the "Exactify!" button to run calculations, the peak table retained old data from previous runs until new results loaded. This created confusion about which results were current.

**Root Cause:** Peak table clearing logic only executed when results were successfully loaded from the output SDF. If processing took time, users saw stale data during execution.

**Fix:** Added immediate peak table clearing at the start of the Exactify run, before the worker thread begins processing.

**Impact:** Peak table now shows a clean slate while processing is underway, providing clearer visual feedback.

---

### 3. ✅ Peak Results Not Populating After Run Completion

**Issue:** After Exactify execution completed, the peak table remained empty instead of displaying the calculated results.

**Root Cause:** The results loading code expected an "EXACT MASS FORMULAS" field in the output SDF that doesn't exist. The actual output format contains only the MASS SPECTRAL PEAKS field with exact m/z values and intensities (no formula data).

**Fix:** Refactored `_load_results_from_sdf()` to:
- Parse the actual output SDF format (exact m/z + intensity pairs)
- Build a map from input nominal m/z to output exact masses
- Populate peak table with matched peaks marked as PASS, unmatched as FAIL
- Correctly handle columns: Input m/z, Intensity, Ion m/z (exact mass), and Filter status
- Leave formula-dependent columns (Fragment Formula, Delta mass, DBE) empty as they're not available in output

**Impact:** Peak table now correctly displays Exactify results with proper input/output peak matching.

## Technical Details

### Output SDF Format Clarification

The output SDF file generated by Exactify contains:
- **MASS SPECTRAL PEAKS field**: Exact m/z and intensity pairs for all matched fragments
- **NO formula data**: Formula assignments are not included in the output file

This means the peak table can display:
- ✅ Input m/z (from input SDF)
- ✅ Input intensity (from input SDF)
- ✅ Ion m/z / exact mass (from output SDF)
- ✅ Filter status (PASS = matched, FAIL = no match)
- ❌ Fragment Formula (not in output SDF)
- ❌ Delta mass (requires formula data)
- ❌ DBE (requires formula data)

## Known Limitations

- Formula assignments and calculated properties (delta mass, DBE) are not available from the Exactify output format
- Peak table can only display exact masses matched to input peaks, not calculated formulas
- These limitations are inherent to the output SDF format, not GUI issues
